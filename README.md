Summary: This project looks to identify patterns of flight delays across the United States airports hoping to understand how geographic location or size may affect delays. In order to accomplish this, two data sets were used. One that contains detailed information about flight delays and another that contains airport passenger traffic data. The first step in the project was for us to clean the data, which ended up being very essential to the process. Both datasets had issues such as missing values, inconsistent formatting, and extreme outliers. We were able to handle missing values by removing rows that had incomplete data allowing for the data to stay consistent. A lot of the numeric values, specifically in the enplanement dataset, were stored as strings with commas or percentage signs, resulting in us converting those values into the proper formats. Additionally, we also noticed some really large delay values which caught our eye. We flagged those and thought about how they might affect the analysis. Overall, cleaning the data made it much easier to work with and helped avoid problems later on.Upon cleaning the dataset, we then merged the datasets. This allowed us to connect delay information with airport characteristics like passenger volume and whether an airport was considered a hub or not. This step was necessary as it made it possible to analyze all the data at once in an organized manner and learn where different correlations existed.After creating out new dataset where all the information was combined and cleaned, we were able to move onto the exploratory data analysis(EDA) portion. In this process, we created several visualizations to understand what the data looked like and what columns connected. These included average delays by region, different types of delays across regions, comparisons between hub and non-hub airports, and delay trends based on airport size. From these graphs, we could already see that there were differences in delays across regions and that larger airports seemed to have higher delays. This gave us a good starting point before running any models overall allowing us to have a better understanding of the data we were working with. Next we moved onto creating two regression models to help answer our two main research questions. The first model looked at how regions affect delays. The coefficients showed that some regions had higher delays than others, but the R² value was extremely low (around 0.004). This told us that the region does not explain much when looking at the reason for the variation in delays. So even though there are slight differences, it is not a strong reliable predictor that can give us information. The second model looked at was based on airport size using passenger volume. This model had a much higher success rate with an R² around 0.81, showing that these factors explain more of the variation in delays. The results suggest that larger airports tend to have higher delays, which correlates since  more traffic results in more complexities when it comes to managing. Overall, the main takeaway from this project is that while delays do vary by region, airport size is a much stronger factor. Bigger airports are generally more prone to delays, likely because of congestion and how busy they are. This gives a better understanding of what actually drives delays and can be useful for both travelers and airports.

Data profile: This project uses two datasets that combined allow us to analyze flight delays across United States airports and allows us to understand how those delays relate to airport size and geographical location. Each dataset provides  different types of information and once we combined them, we were able to get a more complete view of what might be influencing these delays. The first dataset is the Airline Delay Causes dataset, which contains detailed information about flight delays. This dataset includes both the number of delayed flights and the total delay time, which makes it useful for understanding not only the frequency of the delays, but also how severe they are. Some of the key variables include total arrival delay (arr_delay), number of delayed flights (arr_del15), and specific delay categories such as carrier delay, weather delay, NAS delay, security delay, and late aircraft delay. Each row represents aggregated flight activity for a specific airport, airline, and time period, which allows us to compare delay patterns across different locations. The dataset also includes a regional classification variable (RO), which groups airports into geographic regions. This is important because it allows for us to test whether delays change by region and identify any patterns based on location. While the dataset provides strong coverage of delay-related variables, it does not include all possible factors that could influence delays, such as staffing issues or airport capacity. The second dataset is the Enplanement dataset. This dataset provides information about passenger traffic at different airports. The dataset includes total annual passenger counts for 2023 and 2024, percentage changes in passenger volume, airport rankings, and hub classifications. It serves as a measure of airport size and overall activity level. Larger airports typically have more passengers which could lead to higher traffic levels causing congestion and potentially more delays. The hub classification is also useful because it allows us to compare airports based on the role they have in the network of airports. Unlike the delay dataset, this dataset is aggregated at the yearly airport level, which means it does not capture the smaller monthly or seasonal changes in passenger traffic. Overall, this limits the level of detail in the analysis, yet it is still able to provide a strong measure of airport size for comparison purposes. As the last step of preparing the data, the two datasets were merged using airport identifiers (airport from the delay dataset and Locid from the enplanement dataset). This merge allowed us to create a unified dataset that combines the delay metrics with airport characteristics such as passenger volume and hub status. Then, this combined dataset allowed us to directly analyze how geographic region and airport size relate to flight delays, and it served as the foundation for all our visualizations and regression models using the singular dataset. 

Data quality: Data quality was an important part of this project because both datasets came from real-world sources and contained several issues that needed to be addressed before analysis. We started with a full assessment of the data and this helped us identify the problems related to completeness, consistency, accuracy, and overall how we were able to use the dataset. These issues were what guided us to move forward as we cleaned and prepared the data.  One of the main issues we observed in both datasets was completeness. The Airline Delay Causes dataset contained missing values across several columns, including delay counts and delay durations. Similarly, the same issue was present in the Enplanement dataset as it had missing values in certain fields. The missing data has the potential to create problems in analysis, especially in regression models, because it can reduce the number of valid observations or introduce bias if handled incorrectly. To address this, rows with missing values were removed to ensure that all remaining data points were complete. Although this reduced the size of the dataset, it improved the overall reliability and consistency of the data being used. Another major issue we found was dealing with syntax. Several columns that should have been numeric, such as passenger counts and percentage changes, were stored as strings with commas and percentage symbols. These formatting issues made the data unusable for calculations until we went in and corrected this. In order to do this, it required removing non-numeric characters and converting the columns into the proper numeric types. Without this step, any analysis involving passenger volume would not have worked and much more difficult to work with. Outliers were also identified as a potential issue with the data quality. During the EDA process, extremely large delay values were found and appeared unrealistic signifying false or inaccurate results if left. Rather than immediately removing them, we first isolated and examined them to understand the frequency and possible impact. This was imperative as removing outliers without investigation could lead to the removal of extreme cases. Finally, there are limitations related to the source and collection of the data. Both datasets come from government sources, which generally means they are fairly reliable, but they still depend on reports from the airlines and airports. This means there may be inconsistencies or delays in reporting leading to some data that may be incomplete or inaccurate. These limitations were acknowledged when interpreting the results. Overall, the data quality assessment helped identify key issues and guided us through the cleaning process. By being able to address these issues related to missing values, formatting, consistency, and outliers, the final dataset was made more reliable, usable, and easy to work with for both visualization and regression analysis.

We found missing pieces in many different areas of the data set and decided to remove any row that had missing data. Both data sets are massive, so removing one didn't seem like a big deal and would reduce overall bias in our analysis. We removed them using the dropna() function and double checked our work afterward to see if any rows had missing values. This test showed 0 pieces of data. Next we looked at all the ranges within the late__aircraft_delay column. We found a gigantic range of delays being over 90 days. Given how absurd this sounded, we decided to remove any pieces of data with at least 60 days of waiting time. This was our professional judgement, and we chose that time as a compromise between allowing long delays, but anything more than two months felt like a typo to us. Next in the enplanement data we noticed that lots of the rows were put in as strings instead of floats or integers. This makes doing regression on the rows impossible, so we had to change it them to integers. The last thing we had to do was change the hub status within the data. We noticed that some of the airports were missing a hub status. We assumed this just meant the data was not a hub, so we didn't remove the data and instead inserted that they were not hubs. Finally, we merged the data using the Locid and airport functions. This allowed for a seamless merging of data and allowed us to get working on the next steps of our project.

Findings:In order to answer both of our research questions, we used a mixture of visualizations and regression analysis. Firstly, we looked at whether regionality has an impact on flight times. Originally, we used visualizations to answer our question, and based on a first glance, it looked like it played a major role. The Western Pacific and Southwest seemed to have the highest amount of flight delays; however, when we ran our regression analysis, we got different results. We found the R^2 to be 0.004. This is a remarkably low number and shows how little of a factor regionality is compared to others. The other research question we were trying to answer was whether busier airports saw higher delays. Again, the visualizations showed a trend. It seemed like the busier an airport was, the more delays people could expect. The mega airports had more minutes delayed and a larger total number of flights delayed. When we ran our regression analysis, we found that it backed up the visualizations. On the other hand, we thought regionality would play a much larger role than it did.

Future work:This project allowed us to gain useful insights into how flight delays relate to region and airport size. There are several ways this analysis could be expanded and improved in the future. One major improvement could be incorporating weather data. Since weather is one of the most common causes of these delays, and our dataset includes a weather delay variable, it does not fully capture real weather conditions, weather that be storms, snowfall, or temperature. Adding a dataset with real weather information by location, date, and time would help explain why certain regions experience more delays than others or even which circumstances to look out for. For example, regions with frequent storms or severe weather patterns may have higher delays giving us more context to the types of differences we observed. Another area for improvement would be using time-series analysis. Our current analysis does not fully account for how delays change over time, specifically across different seasons. Flight delays are most typically not consistent throughout the year, with higher delays during peak travel seasons such as holidays or summer months as there is more traffic around these times. By analyzing the data over time, we could identify trends, patterns, and seasonal effects that are not visible in the general analysis we have conducted. This would allow for more accurate results and provide a deeper understanding of delay behavior. Additionally, the passenger traffic data we used in our data is aggregated at the yearly level, which limits how detailed our analysis can be and the information we can extract from it. Future work could involve using smaller data, such as monthly or even daily passenger counts. This would allow us to better align passenger volume with delay data, which is readily available at a more detailed level. Overall, having this more precise data would improve the accuracy of our regression models and make the relationship between airport size and delays clearer. In addition to this, we could also expand the model by including additional variables that may impact delays. For example, airport capacity, number of runways, airline-specific performance, and staffing levels could all potentially influence the delay patterns. If we were to include these factors in a more advanced regression model,  it would likely improve the accuracy of the model and provide us with a more complete picture of what causes/ drives the delays. Lastly, future work could explore not only advancing the currency model, but into more advanced modeling techniques. While we used linear regression in this project, other approaches such as multiple regressions or machine learning models could help provide better predictions and help us find the more complex relationships that are present in the data. These methods could help identify non-linear patterns or interactions between variables that are not captured in a simple model or through the EDA process. In conclusion, while this project provides a strong foundation, there is great opportunity to continue strengthening the analysis and gain better insights into the causes of flight delays.
