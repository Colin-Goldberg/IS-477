Since completing our initial project proposal, we have made progress on data cleaning and preparation. This was a big first step as this was one of the most critical steps in our workflow. We began by importing both datasets, the Airline Delay Causes dataset and the Enplanement dataset, into Python using pandas. When initially looking into the airline delay dataset, we were able to identify several missing values across key numerical columns such as arrival delays, delay counts, and delay types. In order to ensure consistency across the and avoid any complications as we continued with our analysis, we removed rows with missing values using (dropna()), resulting in a fully clean dataset with no null entries across all 21 variables. In addition to dealing with missing data, we did exploratory checks on key variables to get a better understanding of their distributions. For example, we examined the range and distribution of the late_aircraft_delay variable and were able to identify extreme outliers. Through this, we were also able to find a small subset of observations with unusually high delay values exceeding 87,600 minutes. These observations were isolated allowing us to understand it better, get a closer look, and determine whether they reflect true extreme cases or potential data quality issues. This step helped us better understand the variability within the dataset and prepare for an analysis later. For the Enplanement dataset, we addressed formatting inconsistencies. We specifically worked with the numerical columns that were stored as strings with commas and percentage signs. We converted columns such as “CY 24 Enplanements,” “CY 23 Enplanements,” and “% Change” into proper numeric formats by removing the non numeric characters and using type conversion functions. We also standardized the Rank column as an integer and handled any missing categorical values by replacing null entries in the Hub column with “Non-hub.” All these steps ensured that all the variables were in cohesive usable formats for analysis. Overall, we successfully cleaned and standardized both datasets, making them easy to use for further analysis. The code used to make these changes are in our repository and will be the foundation of our project as we continue to move onto the next stages of our project.
